WindowSMART Disk Health

WindowSMART takes several variables into consideration when calculating a disk's health:

  • Whether or not any SMART attribute value has fallen less than or equal to its threshold. Even attributes that are Advisory (as opposed to Pre-Fail) can fail if their threshold is greater than zero. Attributes with zero thresholds cannot technically fail.
  • Whether or not any of the super-critical attributes has reached a degraded state (value is still greater than threshold, but Raw Data shows a potential problem). The super-critical attributes are, and they can reach a degraded state when as few as one event is detected (if a threshold is violated, the attribute is set to fail). Items marked with an asterisk (*) are found only on SSDs and vary by SSD controller manufacturer.
    • 05 - Reallocated (Bad) Sector Count (HDD) / Retired Sector Count (SSD)
    • 0A - Spin Retry Count
    • B8 - End-to-End Error (exceedingly rare, but severe)
    • C4 - Reallocation Event Count
    • C5 - Pending Sector Count
    • C6 - (Offline) Uncorrectable Sector Count
    • C7 - Ultra DMA CRC Error Count (least serious of all super-criticals - no starburst icon)
    • E6 - Life Curve Status*
    • E7 - SSD Life Left*
    • E8 - Endurance Remaining*
    • E9 - Available Reserved Space / Media Wearout Indicator*
    • EB - Good Block Rate*
    • FB - Minimum Spares Remaining*
  • Raw Read and Seek Error Rate are also shown in the UI as being super critical but unfortunately these attributes vary widely between disk manufacturers and I haven't found a solid guideline for these and so I don't flag them at this time.
  • Whether or not a disk temperature has reached extreme (critical) or hot (warning) levels.
  • Whether or not an attribute is geriatric.
  • For disks that do not report S.M.A.R.T. data, whether or not the storage driver (WMI) failure prediction flag is set to True.

Home Server SMART specifies the following precedence for disk health:

  1. Critical
  2. Warning
  3. Geriatric
  4. Unknown
  5. Healthy

In other words, if Home Server SMART detects even a single attribute that causes a disk's health to be Critical, it doesn't matter if all of the other attributes are perfectly healthy. The disk is still critical. If there is a combination of Critical, Warning and Healthy, the overall status is still Critical. Put simply, in order to provide the best level of protection (advance warning), the worst status prevails.

A disk will possess a health status of Unknown if the the disk does not report S.M.A.R.T. data. This lack of reporting may be due to a USB bridge chip in an external enclosure that doesn't support it, or it may be because the disk is part of a hardware RAID or attached to a SCSI controller.
