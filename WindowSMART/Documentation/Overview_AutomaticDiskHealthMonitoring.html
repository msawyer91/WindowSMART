Automatic Disk Health Monitoring and Alerting

When you install WindowSMART 2015, a special component called a Windows Service is installed. Windows services are special applications that are designed to run at computer boot-time, and run continuously, whether or not a user is logged on. The WindowSMART 2015 Service starts automatically each time your computer is rebooted, and runs continuously without any intervention on your part.

The Home Server SMART Service polls your disks at a predefined interval, by default every three minutes. You can adjust the interval to as little as once per minute to as high as once per week (10800 minutes). Be careful about using large time intervals. Some people prefer longer intervals to allow their drives to sleep (go into power-save mode) for extended periods, since SMART polling will awaken a sleeping disk. Large intervals, however, could potentially allow a serious problem to fester for days before it is detected and reported.

The service performs four major operations at each polling interval:

  • Collect data about each disk and write it to the Registry. This allows the information to be processed later as well as provides fast access from the UI. This information includes SMART data, whether the disk is an SSD, capacity, etc.
  • Examine the data stored in the Registry to check for disk health problems. If any problems are detected, they are added to the alert message queue. Problems that no longer exist are flagged for removal from the queue.
  • Clear stale disk alerts from the message queue. A disk is considered stale if it has been removed from the computer. WindowSMART does not delete this disk's saved information, since the disk may be added back later. But if there are any active alerts for this disk, they can be cleared since the disk is removed.
  • Process the alert message queue. New problems, as well as existing problems with changes, generate notificaton alerts via email, Prowl and/or NMA, if they are configured. Problems that have cleared, including stale disk events being removed, also will generate a notification indicating the problem has gone away. Existing items with no changes do not generate notifications.
